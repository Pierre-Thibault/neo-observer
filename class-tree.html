Class Hierarchy

  • object: The most base type
    • neo_observer.Event: Encapsulate the information about an event.
    • neo_observer.IObserver: The interface defining the method a class must implement to receive events.
    • neo_observer.ObserverRegistry: The registry containing all the observers.
    • neo_observer._ObserverHolder: An abstract class encapsulating of observer so they all can be treated polymorphically.
      • neo_observer._HardRefObserverHolder: An abstract class for an observer that is a function that can be called with the single parameter event or no parameter at all.
      • neo_observer._IObserverHolder: An abstract final holder for an IObserver to be scan by _ObserverHolder.
      • neo_observer._NullObserverHolder: An null observer just needed to remove observers.
      • neo_observer._WeakRefObserverHolder: An abstract holder using a weak reference.
        • neo_observer._IObserverWeakRefInstanceObserverHolder: An IObserver holder built on top of _WeakRefObserverHolder.
        • neo_observer._OneParamWeakRefObserverHolder: A concrete class of _WeakRefObserverHolder for a one parameter callable.
        • neo_observer._ZeroParamWeakRefObserverHolder: A concrete class of _WeakRefObserverHolder for a zero parameter callable.
    • neo_observer._ObserverRegistryDelegate: An abstract delegate class for ObserverRegistry representing in a type hierarchy of the different type of observer registration.
      • neo_observer._AllEventsObserverRegistryDelegate: A registry for observers who want to be notified of all events.
      • neo_observer._NamesObserverRegistryDelegate: A registry for observers who want to be notified of all events sent with a specific name.
      • neo_observer._SendersAndNamesObserverRegistryDelegate: A registry for observers who want to be notified of all events sent by a specific sender under a certain name.
      • neo_observer._SendersObserverRegistryDelegate: A registry for observers who want to be notified of all events sent by a specific sender.
